# AI Email Assistant

An n8n workflow that monitors a Gmail inbox, classifies incoming emails via GPT-4o-mini, generates a draft reply, and sends it to Telegram for human approval — with one-tap Send / Edit / Reject buttons.

## Features

- Automatic Gmail polling every minute (unread messages only)
- Filters out noreply / no-reply addresses automatically
- Email classification: `inquiry` · `complaint` · `spam` · `newsletter` · `support` · `other`
- Priority detection: `high` · `medium` · `low`
- Draft reply generated by GPT-4o-mini in the original language of the email
- Telegram notification card with summary, category, priority, and draft text
- Inline approval buttons: ✅ Send · ✏️ Edit · ❌ Reject
- On approval — reply is sent directly into the original Gmail thread
- Confirmation message in Telegram after action is executed

## Architecture

```
Gmail Inbox (unread)
  └─ Gmail Trigger (polling · every 60s)
       └─ IF Filter (skip noreply / no-reply)
            └─ HTTP Request → OpenAI GPT-4o-mini
                 └─ Code node (parse JSON response)
                      └─ Telegram: send card + inline buttons
                           └─ Telegram Trigger (webhook · callback_query)
                                └─ Switch
                                     ├─ send   → Gmail Reply → Telegram Confirm ✅
                                     ├─ edit   → Telegram Confirm (dismissed) ✏️
                                     └─ reject → Telegram Confirm (dismissed) ❌
```

### Node reference

| #   | Node             | Type              | Description                                                               |
| --- | ---------------- | ----------------- | ------------------------------------------------------------------------- |
| 1   | Gmail Trigger    | `gmailTrigger`    | Polls INBOX every minute, only unread, no attachments                     |
| 2   | IF Filter        | `if`              | Drops emails where `from` header contains `noreply` or `no-reply`         |
| 3   | OpenAI Request   | `httpRequest`     | POST to `/v1/chat/completions`, model `gpt-4o-mini`, returns JSON         |
| 4   | Parse Response   | `code`            | Extracts `category`, `priority`, `summary`, `draft_reply` from GPT output |
| 5   | Telegram Card    | `telegram`        | Sends HTML message with inline keyboard to your chat                      |
| 6   | Telegram Trigger | `telegramTrigger` | Webhook, listens for `callback_query` events                              |
| 7   | Action Switch    | `switch`          | Routes by callback data: `send:*` · `edit` · `reject`                     |
| 8   | Gmail Send       | `gmail`           | Replies to original thread using message ID from callback data            |
| 9   | Telegram Confirm | `telegram`        | Sends confirmation after action is completed                              |

## Quick Start (local)

```bash
git clone https://github.com/VaalaramaDev/n8n-email-assistant
cd n8n-email-assistant

cp .env.example .env
# Fill in all variables (see Environment Variables section)

docker compose up -d
```

Open n8n at `http://localhost:5678`, complete the setup wizard, then:

1. Import `workflows/email_assistant.json` via **Settings → Workflows → Import from file**

3. Activate the workflow

## Credentials Setup

Configure all credentials in n8n UI under **Settings → Credentials**.

### 1. OpenAI API

- Go to **Settings → Credentials → New → OpenAI API**
- Paste your API key (`sk-...`)
- Save as `OpenAI account`

### 2. Telegram Bot

- Message [@BotFather](https://t.me/BotFather) → `/newbot` → follow prompts
- Copy the bot token
- Go to **Settings → Credentials → New → Telegram API**
- Paste the token, save as `Telegram account`
- Get your chat ID: message [@userinfobot](https://t.me/userinfobot) → copy the `Id:` value
- Set `TELEGRAM_CHAT_ID` in `.env`

### 3. Gmail OAuth 2.0

Full step-by-step guide: [docs/setup-gmail-oauth.md](docs/setup-gmail-oauth.md)

Short version:

1. Open [Google Cloud Console](https://console.cloud.google.com) → create or select a project
2. Enable **Gmail API** (APIs & Services → Library → search Gmail API → Enable)
3. Go to **Google Auth Platform → Audience** → set to External → save
4. Go to **Clients → Create OAuth client**:
   - Application type: **Web application**
   - Name: `n8n`
   - Authorized redirect URI: `https://YOUR_DOMAIN/rest/oauth2-credential/callback`
5. Copy **Client ID** and **Client Secret**
6. Add your Gmail address as a test user (**Audience → Test users → Add users**)
7. In n8n: **Settings → Credentials → New → Gmail OAuth2 API**
   - Paste Client ID and Client Secret
   - Click **Sign in with Google** → authorize
   - Save as `Gmail account`

## Gmail OAuth Setup with ngrok (no domain)

If your n8n runs on a VPS without a domain, Google will reject the IP-based redirect URI. Use ngrok to get a temporary HTTPS URL:

```bash
# Install ngrok
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-amd64.tgz
tar xvzf ngrok-v3-stable-linux-amd64.tgz
sudo mv ngrok /usr/local/bin

# Authenticate (get token at ngrok.com)
ngrok config add-authtoken YOUR_NGROK_TOKEN

# Start tunnel in background
screen -S ngrok
ngrok http 5678
# Note the Forwarding URL, e.g. https://abc123.ngrok-free.dev
# Press Ctrl+A, D to detach
```

Update `.env`:

```env
N8N_HOST=abc123.ngrok-free.dev
N8N_PROTOCOL=https
WEBHOOK_URL=https://abc123.ngrok-free.dev/
N8N_EDITOR_BASE_URL=https://abc123.ngrok-free.dev
N8N_SECURE_COOKIE=true
```

Use `https://abc123.ngrok-free.dev/rest/oauth2-credential/callback` as the redirect URI in Google Cloud Console.

> **Warning:** Free ngrok URLs change on every restart. You will need to update the redirect URI in Google Cloud Console and re-authorize Gmail whenever the tunnel restarts.

## Gmail OAuth Setup with Custom Domain (recommended for production)

Using a real domain with HTTPS is the proper way to run n8n in production. It gives you a stable redirect URI, no dependency on ngrok, and a public URL that works forever.

### Step 1 — Point your domain to the VPS

Add an **A record** in your DNS provider:

| Type | Name                 | Value         |
| ---- | -------------------- | ------------- |
| A    | `n8n.yourdomain.com` | `YOUR_VPS_IP` |

Wait for DNS propagation (usually 5–15 minutes). Verify:

```bash
ping n8n.yourdomain.com
```

### Step 2 — Install Nginx

```bash
apt update && apt install -y nginx
```

### Step 3 — Create Nginx config for n8n

```bash
nano /etc/nginx/sites-available/n8n
```

Paste:

```nginx
server {
    listen 80;
    server_name n8n.yourdomain.com;

    location / {
        proxy_pass http://localhost:5678;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        chunked_transfer_encoding on;
        proxy_buffering off;
        proxy_read_timeout 3600s;
    }
}
```

Enable the config:

```bash
ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

### Step 4 — Issue SSL certificate with Let's Encrypt

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d n8n.yourdomain.com
```

Follow the prompts. Certbot will automatically update the Nginx config for HTTPS and set up auto-renewal.

Verify HTTPS works:

```bash
curl -I https://n8n.yourdomain.com
```

### Step 5 — Update .env

```bash
nano ~/projects/n8n-email-assistant/.env
```

```env
N8N_HOST=n8n.yourdomain.com
N8N_PROTOCOL=https
N8N_PORT=5678
WEBHOOK_URL=https://n8n.yourdomain.com/
N8N_EDITOR_BASE_URL=https://n8n.yourdomain.com
N8N_SECURE_COOKIE=true
```

Restart n8n:

```bash
cd ~/projects/n8n-email-assistant
docker compose down && docker compose up -d
```

### Step 6 — Update Google Cloud Console

1. Open [Google Cloud Console](https://console.cloud.google.com) → **Google Auth Platform → Clients**
2. Click your OAuth client → edit
3. Under **Authorized redirect URIs** — replace the old URL with:
   ```
   https://n8n.yourdomain.com/rest/oauth2-credential/callback
   ```
4. Save

### Step 7 — Re-authorize Gmail in n8n

1. Open n8n at `https://n8n.yourdomain.com`
2. Go to **Settings → Credentials → Gmail account → edit**
3. Click **Sign in with Google** → re-authorize
4. Save

The workflow will continue working with the same credential name — no changes needed inside the workflow nodes.

> **Auto-renewal:** Let's Encrypt certificates expire every 90 days. Certbot installs a cron job that renews automatically. To test renewal: `certbot renew --dry-run`

## Environment Variables

| Variable              | Description                                     | Example                               |
| --------------------- | ----------------------------------------------- | ------------------------------------- |
| `N8N_USER`            | n8n basic auth username                         | `admin`                               |
| `N8N_PASSWORD`        | n8n basic auth password                         | `strongpassword`                      |
| `N8N_HOST`            | Public hostname (no protocol, no port)          | `178.104.202.207` or `yourdomain.com` |
| `N8N_PROTOCOL`        | `http` or `https`                               | `https`                               |
| `N8N_PORT`            | Internal port                                   | `5678`                                |
| `N8N_EDITOR_BASE_URL` | Full public URL of n8n editor                   | `https://yourdomain.com`              |
| `WEBHOOK_URL`         | Full public URL for webhooks (trailing slash)   | `https://yourdomain.com/`             |
| `N8N_ENCRYPTION_KEY`  | Random 32-char string for credential encryption | `openssl rand -hex 16`                |
| `N8N_SECURE_COOKIE`   | Set to `false` for plain HTTP                   | `true`                                |
| `OPENAI_API_KEY`      | OpenAI API key                                  | `sk-proj-...`                         |
| `TELEGRAM_BOT_TOKEN`  | Telegram bot token from @BotFather              | `123456:ABC-DEF...`                   |
| `TELEGRAM_CHAT_ID`    | Your Telegram user ID                           | `8409896342`                          |

> Gmail OAuth credentials (Client ID and Client Secret) are configured in the n8n credentials UI — never stored in `.env`.

## Known Limitations

- **ngrok free tier:** Public URL changes on every tunnel restart. For a stable setup, use a custom domain with Nginx + Let's Encrypt.
- **Telegram callback data limit:** Telegram limits `callback_data` to 64 bytes. The current implementation passes only the Gmail message ID in the callback and extracts the draft reply from the message text — this keeps the payload small.
- **Gmail OAuth app verification:** The OAuth app runs in "Testing" mode, which means only explicitly added test users can authorize it. This is sufficient for personal and portfolio use.
- **Polling delay:** Gmail is checked every 60 seconds, so notification delivery may be delayed by up to 1 minute.
